# Структура открытого репозитория российской робототехники

Документ описывает организацию репозитория, в котором для каждой известной модели робота представлено русскоязычное программное обеспечение.

---

## 1. Цель

Создать единый опенсорс-репозиторий, где:

- Для каждой поддержанной модели робота — свой каталог с ПО
- Весь код, документация и комментарии — на русском языке
- Общий SDK и утилиты переиспользуются между моделями
- Сообщество может добавлять поддержку новых моделей

---

## 3. Структура каталога одной модели

Для каждой модели робота — единообразная структура:

```
models/<model_id>/
├── README.md              # Описание модели, производитель, характеристики
│                         # Ссылки на железо, BOM, где купить
│                         # Требования (ROS2, ОС, оборудование)
├── CHANGELOG.md           # История изменений ПО для этой модели
│
├── driver/                # Драйверы и низкоуровневый код
│   ├── README.md          # Архитектура драйвера
│   ├── src/               # Исходники (Python, C++, Rust)
│   ├── include/           # Заголовки (если C++)
│   └── tests/             # Тесты драйвера
│
├── config/                # Конфигурации
│   ├── default.yaml       # Базовые параметры
│   ├── calibration/       # Файлы калибровки
│   └── urdf/              # URDF, меши (если применимо)
│
├── docs/                  # Документация на русском
│   ├── setup.md           # Установка и настройка
│   ├── api.md             # API драйвера
│   ├── calibration.md    # Процедура калибровки
│   └── troubleshooting.md
│
└── scripts/               # Скрипты
    ├── install.sh         # Установка зависимостей
    ├── run_sim.sh         # Запуск в симуляции
    ├── run_hw.sh          # Запуск на железе
    └── calibrate.sh       # Калибровка
```

## 4. Содержимое README модели

Каждый `models/<model_id>/README.md` содержит:

| Раздел | Содержание |
|--------|-------------|
| Название и производитель | Модель, страна, год выпуска |
| Характеристики | Степени свободы, датчики, грузоподъёмность, автономность |
| Железо | Ссылки на спецификации, BOM, где купить/заказать |
| Требования | ROS2-версия, ОС, доп. оборудование |
| Быстрый старт | 3–5 команд для первого запуска |
| Статус поддержки | Полная / частичная / экспериментальная |
| Известные ограничения | Что пока не работает |

## 6. Правила именования

- **model_id** — строчные буквы, цифры, подчёркивание: `atom01`, `unitree_h1`, `xiaomi_cyberone`
- **Ветки** — `model/<model_id>` для разработки по модели
- **Теги** — `v1.0.0`, `atom01/v0.2.0` (версия модели)

---

## 7. Требования к русскоязычному ПО

- Все README, CONTRIBUTING, документация — на русском
- Docstrings и комментарии в коде — на русском
- Сообщения коммитов — желательно на русском
- Имена переменных и функций — на английском (стандарт программирования)
- Пользовательские сообщения (логи, ошибки, подсказки) — на русском

---

## 8. Процесс добавления новой модели

1. Копировать `models/template/` в `models/<model_id>/`
2. Заполнить README модели
3. Реализовать драйвер (адаптер к SDK)
4. Добавить конфиги и документацию
5. Пройти проверку: сборка, тесты, базовый сценарий
6. Оформить Pull Request с описанием модели

---

## 9. Лицензирование

- **SDK и общий код** — GPL v3 или Apache 2.0 (единообразно)
- **Драйверы моделей** — та же лицензия, что и SDK
- **Сторонний код** — сохранять оригинальные лицензии, указывать в NOTICE

---

## 10. Связь с другими компонентами

| Компонент | Связь с репозиторием |
|-----------|----------------------|
| **OpenClaw** | SDK предоставляет API для Skill «управление роботом»; агент генерирует код под выбранную модель |
| **roboto_origin** | Atom01 — первая полноценно поддержанная модель |
| **CVCODE / ANIEMORE** | Модули в корне репо, интеграция через SDK |
