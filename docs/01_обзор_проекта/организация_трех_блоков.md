# Организация трёх блоков проекта

Документ описывает, как организовать и связать три основных блока проекта национальной платформы робототехники: материальная база, софт и образование в колледжах/техникумах.

---

## 1. Общая схема взаимодействия блоков

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│                        ВЗАИМОСВЯЗЬ ТРЁХ БЛОКОВ                                       │
├─────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                      │
│   ┌──────────────────────┐     ┌──────────────────────┐     ┌───────────────────┐  │
│   │  БЛОК 1               │     │  БЛОК 2               │     │  БЛОК 3          │  │
│   │  Материальная база    │────▶│  Разработка софта     │◀────│  Образование      │  │
│   │  робота               │     │                       │     │  (колледжи)        │  │
│   └───────────┬──────────┘     └───────────┬───────────┘     └─────────┬─────────┘  │
│               │                            │                         │             │
│               │  Железо для тестов          │  SDK, API для железа    │  Выпускники │
│               │  и пилотов                  │                         │  в блоки 1,2│
│               │                            │                         │             │
│               └────────────────────────────┴─────────────────────────┘             │
│                                        │                                           │
│                                        ▼                                           │
│                              Пилоты, внедрения,                                    │
│                              серийное производство                                  │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

**Принцип:** Блоки работают параллельно, но согласовано. Материальная база даёт железо для тестов софта и для учебных стендов. Софт требует реальных роботов для отладки. Образование поставляет кадры в оба блока и потребляет их результаты (роботы + SDK) как учебный контент.

---

## 2. Блок 1: Разработка материальной базы робота

### 2.1. Цель

Создать и поддерживать физическую основу робототехники: компоненты, сборку, ремонт, номенклатуру.

### 2.2. Структура работ

| Подблок | Содержание | Результат |
|---------|------------|-----------|
| **Закупка и логистика** | Импорт образцов (Unitree, DeepRobotics, Dobot и др.), доставка, таможня | Парк роботов для R&D и пилотов |
| **Реверс-инжиниринг** | Разборка, чертежи, протоколы, BOM | Документация, возможность локализации |
| **Каталогизация** | Компонентная база, поставщики (см. [компонентная_база_и_поставки.md](компонентная_база_и_поставки.md)) | Каталог с российскими аналогами |
| **Сборочная мастерская** | Оборудование, процессы, ЗИП, ремонт | Готовность к сборке и сервису |
| **Локализация** | Замена импортных компонентов на российские (DMOTOR, Rozum, РАСМО и др.) | Импортозамещённая номенклатура |
| **Производство (Э2)** | Линии сборки, производство компонентов | Серийное производство в Москве |

### 2.3. Организационная форма

- **Этап 1:** Экспериментальная лаборатория — площадка в Москве (или Солнечногорск по цепочкам поставок)
- **Этап 2:** Завод с цехами компонентов и сборки

### 2.4. Связь с другими блоками

| Куда передаёт | Что передаёт |
|----------------|--------------|
| → Блок 2 (Софт) | Реальные роботы для тестов SDK, образцы для драйверов, протоколы |
| → Блок 3 (Образование) | Учебные роботы, разобранные узлы, комплекты для практики |

| Откуда получает | Что получает |
|-----------------|--------------|
| ← Блок 2 | Требования к железу, обратная связь по прошивкам |
| ← Блок 3 | Стажёры, выпускники — сборщики, техники, электромеханики |

---

## 3. Блок 2: Разработка софта

### 3.1. Цель

Создать программную платформу для управления роботами: SDK, LLM-агент, интеграции.

### 3.2. Структура работ

| Подблок | Содержание | Результат |
|---------|------------|-----------|
| **SDK** | API, драйверы для 2–3 моделей, документация на русском | Открытый SDK (см. [структура_открытого_репозитория.md](../02_робот/структура_открытого_репозитория.md)) |
| **LLM-агент** | ТЗ на естественном языке → алгоритм движения (OpenClaw и др.) | MVP агента |
| **Интеграция** | MES/SCADA/ERP, CVCODE, ANIEMORE | API для предприятий |
| **Low-code** | Визуальный конструктор сценариев | Инструмент для настройки без программирования |
| **Мониторинг** | Диагностика, предиктивное обслуживание | Сервисная платформа |

### 3.3. Организационная форма

- Команда разработки (Python, C++, ROS2)
- Репозиторий с открытой структурой по моделям роботов
- CI/CD, тесты на реальном железе из блока 1

### 3.4. Связь с другими блоками

| Куда передаёт | Что передаёт |
|----------------|--------------|
| → Блок 1 | Требования к железу, протоколам, разъёмам |
| → Блок 3 | SDK как учебный материал, задачи для студентов |

| Откуда получает | Что получает |
|-----------------|--------------|
| ← Блок 1 | Роботы для разработки и отладки |
| ← Блок 3 | Стажёры, junior-разработчики |

## 4. Блок 3: Образование в колледжах и техникумах

### 4.1. Цель

Создать и развить систему подготовки специалистов среднего звена: техники, сборщики, операторы, программисты робототехники.

### 4.2. Контекст в России

- **Специальность СПО 15.02.10** «Мехатроника и робототехника (по отраслям)» — более 138 колледжей
- **WorldSkills** — компетенции «Промышленная робототехника», «Мобильная робототехника»
- Проблема: часто нет современного оборудования и актуальных учебных программ под гуманоидов и новую робототехнику

### 4.3. Два подхода к организации

#### Вариант А: Партнёрство с существующими колледжами

| Шаг | Действие |
|-----|----------|
| 1 | Выбрать 3–5 колледжей в Москве и МО с профилем 15.02.10 или мехатроника |
| 2 | Заключить соглашения о сотрудничестве |
| 3 | Поставить учебные роботы (из блока 1) или разборные стенды |
| 4 | Внедрить модули по SDK (из блока 2) в программу |
| 5 | Организовать стажировки на площадке блока 1 |
| 6 | Участие в WorldSkills, олимпиады |

#### Вариант Б: Создание специализированных образовательных центров

| Шаг | Действие |
|-----|----------|
| 1 | Инициировать создание «Колледжа робототехники» (или аналогичного названия) |
| 2 | Формат: отдельная площадка или структурное подразделение при вузе/предприятии |
| 3 | Лицензия на СПО, аккредитация по 15.02.10 и смежным специальностям |
| 4 | Учебные программы под платформу проекта: железо + софт |
| 5 | Мастерские с реальными роботами, сборочная зона |

### 4.4. Структура образовательного блока

| Подблок | Содержание | Результат |
|---------|------------|-----------|
| **Программы** | Адаптация ФГОС под платформу, модули по SDK и сборке | Учебные планы, методички |
| **Мастерские** | Оборудование: роботы, стенды, компоненты (из блока 1) | Рабочие места для практики |
| **Контент** | Документация SDK, видео, задачи (из блока 2) | Учебные материалы на русском |
| **Стажировки** | Практика в лаборатории/заводе (блок 1) и в команде софта (блок 2) | Трудоустройство выпускников |
| **Соревнования** | WorldSkills, региональные олимпиады | Популяризация, отбор талантов |
| **Партнёрская сеть** | Колледжи в Москве, МО, регионах | Масштабирование подготовки |

### 4.5. Специализированные направления в колледжах

| Направление | Квалификация | Связь с блоками 1 и 2 |
|-------------|--------------|------------------------|
| **Сборка и обслуживание** | Слесарь-сборщик, техник-мехатроник | Блок 1: мастерская, ЗИП |
| **Программирование** | Программист робототехники | Блок 2: SDK, драйверы |
| **Оператор/интегратор** | Специалист по внедрению | Оба: роботы + софт |
| **Сервис и ремонт** | Специалист по ремонту роботов | Блок 1: диагностика, замена узлов |

### 4.6. Связь с другими блоками

| Куда передаёт | Что передаёт |
|----------------|--------------|
| → Блок 1 | Выпускники-сборщики, техники; обратная связь по учебным комплектам |
| → Блок 2 | Выпускники-программисты; предложения по улучшению документации |

| Откуда получает | Что получает |
|-----------------|--------------|
| ← Блок 1 | Роботы, стенды, компоненты, стажировки |
| ← Блок 2 | SDK, учебные материалы, стажировки в команде разработки |

## 5. Координация трёх блоков

### 5.1. Управляющая структура

| Механизм | Описание |
|----------|----------|
| **Единый проект** | Один проект с тремя блоками, общий бюджет и дорожная карта |
| **Регулярные созвоны** | Руководители блоков — еженедельно/двухнедельно |
| **Совместные стенды** | Лаборатория как общая площадка для блоков 1 и 2; колледжи — партнёры блока 3 |
| **Единый репозиторий** | Структура по моделям: железо (BOM, чертежи) + софт (драйверы) + docs (учебные материалы) |

### 5.2. Синхронизация по этапам

| Этап | Блок 1 | Блок 2 | Блок 3 |
|------|--------|--------|--------|
| **Э1 (1 млрд)** | Лаборатория, импорт, каталог, 1–3 прототипа | SDK MVP, драйверы 2–3 моделей, LLM-агент | Переговоры с колледжами, первые соглашения, пилотная группа стажёров |
| **Э2 (9 млрд)** | Производство, локализация | Полный SDK, low-code, интеграции | Специализированные колледжи/центры, массовые стажировки, WorldSkills |

### 5.3. Критерии согласованности

- Блок 1 поставляет железо не позднее, чем блок 2 начинает разработку драйвера под модель
- Блок 3 получает учебные материалы (документация SDK) одновременно с релизом для пилотов
- Выпускники блока 3 трудоустраиваются в блоки 1 и 2 — KPI по целевым показателям

---

## 6. Риски и меры

| Риск | Мера |
|------|------|
| Блок 1 отстаёт — нет железа для софта | Параллельная закупка импортных образцов и старт реверс-инжиниринга |
| Колледжи не заинтересованы | Гранты на оборудование, гарантии трудоустройства, статус пилотной площадки |
| Нет преподавателей под новую платформу | Обучение преподавателей на базе блоков 1–2, приглашение практиков |
| Разрыв между учебными программами и реальным производством | Модули разрабатываются совместно с блоком 1 и 2 |
