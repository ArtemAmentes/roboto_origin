# Хоппер-кун — балансирующий робот (обратный маятник)

> Источник: [https://watako-lab.com/2023/02/19/hopper-kun/](https://watako-lab.com/2023/02/19/hopper-kun/)

## Обзор проекта

**Хоппер-кун** (ホッパー君) — самодельный двухколёсный балансирующий робот, работающий по принципу обратного маятника. Способен к автономному перемещению, обходу препятствий, отслеживанию объектов и распознаванию лиц/объектов.

### Ключевые возможности

- Балансировка на двух колёсах (обратный маятник)
- Автономная навигация с планированием маршрута (A*)
- Обнаружение и отслеживание объектов по цвету и CNN
- Распознавание лиц
- Голосовое управление через Android
- Движение по наклонным поверхностям
- Прыжки

## Конструкция

### Механика ног

- Механизм для прыжков — тестирование показало высоту ~1 см
- Исследование достаточности крутящего момента моторов
- Дизайн в форме птицы с большой головой

### Голова

Содержит:
- 3 ToF-дальномера
- Камеру для CNN-детектирования
- Сервоприводы наклона (pitch) и поворота (yaw) головы
- Динамик для воспроизведения звука

### Аппаратная платформа

- **3 микроконтроллера ESP32**:
  - ESP32 #1 — «мозг» (распознавание, подобие SLAM)
  - ESP32 #2 — управление движением (обратный маятник, одометрия)
  - M5Atom — вспомогательные функции
- MAixM1n для CNN-распознавания объектов
- Контроллер VS-C3 для беспроводного управления

## Управление обратным маятником

### Первый тест балансировки

Удалённое управление контроллером. Грубое PID-управление по углу тангажа — и даже это работает достаточно хорошо.

### Свободное перемещение

Реализовано свободное движение при балансировке + независимое перемещение головы. Учтена компенсация момента инерции головы и смещения центра масс при её повороте.

### Подъём из лежачего положения

Реализовано автономное вставание и переход в балансировочный режим.

### Движение по наклонной поверхности

Стабильность езды значительно улучшена за счёт алгоритмов стабилизации и настройки параметров.

## Навигация и планирование пути

### Алгоритм A* (поиск пути)

- Реализован алгоритм A* на ESP32
- Сравнение fScore vs hScore (только эвристика) — hScore не даёт оптимальный путь, но снижает вычислительные затраты

### Преследование по траектории (PurePursuit)

- Моделирование как дифференциальный двухколёсный робот
- Алгоритм PurePursuit для следования по вычисленному пути
- Робот может поворачиваться на месте, поэтому отклонения от пути минимальны

### Построение карты (ToF + одометрия)

- ToF-сенсоры для обнаружения стен
- Одометрия по колёсам для оценки собственного положения
- Визуализация в реальном времени на ПК
- Накопление ошибок одометрии — планируется изучение SLAM

### Планирование пути в реальном времени

- 3 ToF-сенсора обнаруживают препятствия
- Маршрут (жёлтая сетка) вычисляется в реальном времени на ESP32
- Результаты передаются на ПК для визуализации

### Экстренное уклонение

При внезапном появлении препятствия или если робот «засмотрелся» и едет на преграду — срабатывает алгоритм экстренного уклонения.

## Система распознавания

### Отслеживание по цвету

- Камера отслеживает объект заданного цвета
- Голова следит за объектом, сохраняя баланс
- Добавлена информация о глубине (ToF) для 3D-локализации

### Распознавание лиц

- MAixM1n для детектирования лиц
- ~11 fps при работе с распознаванием
- Робот поворачивает голову, следя за лицом — выглядит «живым»

### CNN-распознавание объектов

- Обучена нейросеть для обнаружения конкретных роботов
- Применение: автономное преследование другого робота

### Распознавание меток

Двухуровневая детектирование: отдельно «голова» и «тело» — только при вертикальном расположении считается целевым маркером.

### Голосовое управление (через Android)

- Android-устройство используется для распознавания речи
- Ключевое слово «Хоппер-кун» активирует диалог
